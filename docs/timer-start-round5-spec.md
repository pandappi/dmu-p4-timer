# 디버프 타이머 수정 작업 요청서

## 목표

기존에는 각 차수마다 디버프를 선택하는 순간 타이머를 시작하는 구조였으나, 이제는 **5차 디버프를 선택하는 순간부터 전체 타임라인이 흘러가도록 수정**한다.

1차~4차에서는 타이머를 시작하지 않고, 사용자가 선택한 정보만 저장한다.
5차 디버프를 선택하고 입력을 완료하는 순간을 `타이머 시작 시점`으로 삼는다.

또한 `마안`은 사용자가 직접 선택하지 않는다.
마안은 1차/3차에 발생하는 공통 기믹으로 처리하며, 해당 차수의 진실/거짓 값에 따라 처리 문구만 자동으로 계산한다.

---

## 핵심 변경사항

### 기존 방식

```text
1차 선택 → 1차 타이머 시작
2차 선택 → 2차 타이머 시작
3차 선택 → 3차 타이머 시작
4차 선택 → 4차 타이머 시작
5차 선택 → 5차 타이머 시작
```

### 변경 방식

```text
1차 선택 → 정보만 저장
2차 선택 → 정보만 저장
3차 선택 → 정보만 저장
4차 선택 → 정보만 저장
5차 선택 → 정보 저장 + 전체 타이머 시작
```

타이머 기준점은 다음과 같다.

```text
5차 디버프 선택 완료 시점 = T+0초
```

---

## 입력 구조

## 1차 입력

1차에서는 다음 정보만 받는다.

```text
- 진실 / 거짓
- 물/번개 시간: 빠른 시간 or 느린 시간
```

물/번개 시간은 파티 전체 기준으로 동일하다.

```text
1차 물/번개 빠른 시간 = 기존 51초 웨이브
1차 물/번개 느린 시간 = 기존 76초 웨이브
```

단, 5차 기준 타임라인에서는 다음처럼 표시한다.

```text
빠른 물/번개/가속도폭탄 처리 = T+20초
느린 물/번개/가속도폭탄 처리 = T+45초
```

### 1차에서 마안은 선택하지 않는다

마안은 1차 공통 기믹으로 자동 등록한다.

```text
1차 마안 처리 = T+29초
```

처리법은 1차의 진실/거짓을 따른다.

```text
1차 진실 → 뒤돌기
1차 거짓 → 마안보기
```

---

## 2차 입력

2차에서는 다음 정보를 받는다.

```text
- 진실 / 거짓
- 혼돈의 불 or 혼돈의 물
```

한글/영문 매핑은 다음과 같다.

```text
혼돈의 불 = Entropy
혼돈의 물 = Dynamic Fluid
```

처리법은 다음과 같다.

```text
혼돈의 불:
- 진실: 나가기
- 거짓: 가만히

혼돈의 물:
- 진실: 가만히
- 거짓: 나가기
```

단, 실제 전체 타임라인에서는 디버프가 2차에 나오든 4차에 나오든 처리 웨이브는 고정된다.

```text
혼돈의 불 처리 = T+36초
혼돈의 물 처리 = T+59초
```

---

## 3차 입력

개인용 Assist에서 3차 입력은 1차에 받은 디버프에 따라 달라진다.

```text
- 진실 / 거짓
- 1차 물/번개 대상자 → 3차 가속도폭탄 시간만 선택
- 1차 가속도폭탄 대상자 → 3차 물 or 번개 + 시간 선택
```

3차 시간 선택은 부여 당시 지속시간 기준으로 받는다.

```text
3차 빠른 패턴 = 36초
3차 느린 패턴 = 61초
```

단, 5차 기준 타임라인에서는 결과적으로 다음 웨이브에 합쳐진다.

```text
빠른 처리 웨이브 = T+20초
느린 처리 웨이브 = T+45초
```

### 3차에서 마안은 선택하지 않는다

마안은 3차 공통 기믹으로 자동 등록한다.

```text
3차 마안 처리 = T+53초
```

처리법은 3차의 진실/거짓을 따른다.

```text
3차 진실 → 뒤돌기
3차 거짓 → 마안보기
```

---

## 4차 입력

4차에서는 다음 정보만 받는다.

```text
- 진실 / 거짓
```

4차 디버프 종류는 2차 선택값으로 자동 계산한다.

```text
2차 혼돈의 불 → 4차 혼돈의 물
2차 혼돈의 물 → 4차 혼돈의 불
```

처리 웨이브는 다음처럼 고정된다.

```text
혼돈의 불 처리 = T+36초
혼돈의 물 처리 = T+59초
```

즉 4차에서는 디버프 종류를 다시 고를 필요가 없다.

---

## 5차 입력

5차에서는 다음 정보를 받는다.

```text
- 알라그필드 or 죽음초월
- 죽은자의 상처 / 파랑 or 산자의 상처 / 보라
```

영문/한글/색상 매핑은 다음과 같다.

```ts
blackWound: {
  key: 'blackWound',
  englishName: 'Black Wound',
  koreanName: '죽은자의 상처',
  shortLabel: '파랑',
  note: '파랑',
}

whiteWound: {
  key: 'whiteWound',
  englishName: 'White Wound',
  koreanName: '산자의 상처',
  shortLabel: '보라',
  note: '보라',
}
```

5차 입력이 완료되면 Assist를 시작한다.

```text
5차 입력 완료 = T+0초
```

5차 처리 타이밍은 다음과 같다.

```text
5차 조합 처리 = T+11초
```

처리법은 다음과 같다.

```text
죽음초월 + 죽은자의 상처 / 파랑 → 파랑 맞기
죽음초월 + 산자의 상처 / 보라 → 보라 맞기

알라그필드 + 죽은자의 상처 / 파랑 → 보라 맞기
알라그필드 + 산자의 상처 / 보라 → 파랑 맞기
```

---

# 5차 기준 전체 타임라인

5차 디버프 선택 완료 시점을 `T+0초`로 잡는다.

|    시간 | 처리 항목            | 표시 문구                |
| ----: | ---------------- | -------------------- |
| T+11초 | 5차 조합 처리         | 파랑 맞기 / 보라 맞기        |
| T+20초 | 빠른 물/번개/가속도폭탄 처리 | 쉐어 / 산개 / 가만히 / 움직이기 |
| T+29초 | 1차 마안 처리         | 뒤돌기 / 마안보기           |
| T+36초 | 혼돈의 불 처리         | 나가기 / 가만히            |
| T+45초 | 느린 물/번개/가속도폭탄 처리 | 쉐어 / 산개 / 가만히 / 움직이기 |
| T+53초 | 3차 마안 처리         | 뒤돌기 / 마안보기           |
| T+59초 | 혼돈의 물 처리         | 가만히 / 나가기            |

---

## 액션 텍스트 규칙

디버프 시간이 완료되면 **디버프명이 아니라 처리법 텍스트를 표시**한다.

예시:

```text
Forked Lightning 만료 → 산개 또는 쉐어
Compressed Water 만료 → 쉐어 또는 산개
Acceleration Bomb 만료 → 가만히 또는 움직이기
Entropy 만료 → 나가기 또는 가만히
Dynamic Fluid 만료 → 가만히 또는 나가기
마안 처리 → 뒤돌기 또는 마안보기
5차 처리 → 파랑 맞기 또는 보라 맞기
```

알림 문구에는 시간 표현을 붙이지 않는다.

```text
나쁜 예:
5초 뒤 나가기
3초 뒤 산개

좋은 예:
나가기
산개
가만히
뒤돌기
마안보기
파랑 맞기
보라 맞기
```

# 5차 조합 처리 함수

```ts
function getFinalAction(
  finalDebuff: FinalDebuff,
  woundDebuff: WoundDebuff
): string {
  if (finalDebuff === 'beyondDeath') {
    return woundDebuff === 'blackWound' ? '파랑 맞기' : '보라 맞기';
  }

  return woundDebuff === 'blackWound' ? '보라 맞기' : '파랑 맞기';
}
```

---

# 마안 처리 함수

```ts
function getEyeAction(truthState: TruthState): string {
  return ACTION_TEXT_MAP.cursedShriek[truthState];
}
```

마안 타이머는 별도 선택 없이 자동 생성한다.

```ts
const round1EyeTimer = {
  seconds: 29,
  actionText: getEyeAction(round1Truth),
};

const round3EyeTimer = {
  seconds: 53,
  actionText: getEyeAction(round3Truth),
};
```

사용자가 5차 선택을 완료하고 `Assist 시작`을 누르면 전체 타임라인이 시작된다.

# 완료 조건

* 1차~4차 입력에서는 타이머가 시작되지 않아야 한다.
* 5차 디버프 선택 완료 시점부터 전체 타이머가 시작되어야 한다.
* 마안은 선택 UI를 제공하지 않아야 한다.
* 1차 마안은 5차 기준 T+29초에 자동 표시되어야 한다.
* 3차 마안은 5차 기준 T+53초에 자동 표시되어야 한다.
* 마안 처리 문구는 각 차수의 진실/거짓 값을 따라야 한다.
* 디버프 만료 시 디버프명이 아니라 처리법 텍스트를 보여줘야 한다.
* 알림 문구에 `5초 뒤`, `3초 뒤` 같은 시간 표현을 붙이지 않아야 한다.
* 5차 조합 처리는 T+11초에 표시되어야 한다.
* 혼돈의 불은 T+36초, 혼돈의 물은 T+59초에 표시되어야 한다.
